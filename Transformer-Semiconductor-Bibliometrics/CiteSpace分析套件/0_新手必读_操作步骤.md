# 新手必读——CiteSpace 七张图操作步骤

> 看完这个文件你就能动手做了。每一步都说得很细，照着点就行。

---

## 你需要准备什么

- 一台装了 CiteSpace 6.4.1 的电脑
- 这个文件夹里的所有文件（别删也别改名）
- 大约 1-2 小时时间

---

## 第一步：导入数据（只做一次）

1. 打开 CiteSpace，点击顶部菜单 **Data → Import/Export**
2. 在弹出的窗口里，点 **WoS** 标签页
3. **Input Directory**：选择这个文件夹（里面有 `1_原始文献数据_第1批.txt` 和 `2_原始文献数据_第2批.txt`）
4. **Output Directory**：选一个空文件夹（比如桌面上新建一个叫 `citespace_output` 的文件夹）
5. 点击 **Remove Duplicates (WoS)** 按钮（去重）
6. 看到 "Finished" 就是成功了，关掉这个窗口

---

## 第二步：创建项目（每张图都要做一遍）

1. 点击顶部菜单 **New** 按钮
2. 弹窗里填：
   - **Title**：随便填，比如"关键词共现"
   - **Project Home**：选刚才新建的 `citespace_output` 文件夹
   - **Data Source**：选 `citespace_output` 文件夹下的 `data` 子文件夹
3. 点 **Save** 保存

---

## 第三步：设置参数（七张图通用）

点击项目面板右下角的 **GO!** 按钮旁边的齿轮图标，或者直接在项目面板里设置：

| 英文项 | 设成什么 | 怎么找 |
|---|---|---|
| Time Slicing | 2015 到 2025，下面填 1 | From 填 2015，To 填 2025，#Years Per Slice 填 1 |
| Node Types | 每张图不同，看下文 | 在中间偏左的位置 |
| Selection Criteria | g-index，k 填 25 | 中间位置，下拉选 g-index，右边框填 25 |
| Pruning | 两个都勾上 | 勾上 Pathfinder，勾上 Pruning sliced networks |

---

## 第四步：七张图分别怎么做

### 图 1：关键词共现网络

| 你要干什么 | 怎么做 |
|---|---|
| 节点类型 | Node Types 选 **Keyword** |
| 点击 | 点 **GO!** 按钮，等它跑完 |
| 调整好看 | 左上角菜单：**Visualization → Network Visualization**，进去后左边面板调颜色、标签大小 |
| 聚类 | 在可视化窗口点 **K** 按钮（聚类），选 LLR 算法，点 OK |
| 需要看到的效果 | 图里节点越多越大越重要，不同颜色块代表不同聚类，每个色块上有文字标签 |
| 导出图片 | **File → Save as PNG**，文件名填 `keyword_cooccurrence_network.png` |
| 验收 | 左上角会有 Modularity Q 和 Silhouette 数值，Q > 0.3 且 Silhouette > 0.7 就合格 |

### 图 2：关键词聚类图

| 你要干什么 | 怎么做 |
|---|---|
| 在图 1 的基础上 | 图 1 做好后，点 Cluster View（聚类视图） |
| 需要看到的效果 | 一个大圆饼分不同色块，#0 到 #4 至少 5 个聚类，每个有文字标签 |
| 导出图片 | **File → Save as PNG**，文件名填 `keyword_cluster.png` |

### 图 3：共被引网络

| 你要干什么 | 怎么做 |
|---|---|
| 返回项目面板 | 别关掉，重新设置参数 |
| 节点类型 | Node Types 选 **Reference** |
| 点击 | 点 **GO!**，等跑完 |
| 调整 | 进可视化，标签显示作者和年份（左边面板 Labels → By Cited Reference） |
| 需要看到的效果 | 节点上是人名+年份（如 Vaswani A, 2017），大节点是被引用多的文献，连线表示被同一篇文章引用过 |
| 导出图片 | `co_citation_network.png` |

### 图 4：共被引聚类图

| 你要干什么 | 怎么做 |
|---|---|
| 在图 3 的基础上 | 点 K 聚类，LLR 算法 |
| 需要看到的效果 | 不同的文献类别用不同色块区分，每个色块上自动生成聚类标签 |
| 导出图片 | `co_citation_cluster.png` |

### 图 5：突现检测时间线

| 你要干什么 | 怎么做 |
|---|---|
| 节点类型 | 选 **Keyword** |
| 跑完后 | 在可视化窗口，顶部菜单选 **Display → Citation Bursts** |
| 排序 | 选 Sort by the beginning year of burst |
| 最小强度 | 设为 1.0（右上角输入框） |
| 需要看到的效果 | 横向蓝色条+红色条，红色段表示突现区间，每条前面有年份和强度数字 |
| 导出图片 | `burst_timeline.png` |

### 图 6：聚类时间线图

| 你要干什么 | 怎么做 |
|---|---|
| 节点类型 | 选 **Keyword**，GO 跑完 |
| 在可视化窗口 | 顶部菜单 **Display → Timeline View** |
| 需要看到的效果 | 横轴是年份（2015-2025），纵轴是聚类编号（#0-#4），每个聚类是一条横向时间轴，上面的圆圈是文献 |
| 导出图片 | `timeline_view.png` |

### 图 7：发文趋势图（用 Excel 做，不是 CiteSpace）

| 你要干什么 | 怎么做 |
|---|---|
| 打开数据 | 双击 `13_发文趋势数据.csv`（Excel 打开） |
| 选中 | 年份 + 发文量两列数据 |
| 插入图表 | 柱状图 |
| 配色 | 2015-2019 灰色、2020-2023 蓝色、2024-2025 黑色 |
| 加趋势线 | 右键柱子 → 添加趋势线，或再加一条折线表示增长率 |
| 标注 | 在 2020 年和 2024 年柱子上面分别加文字"ViT 提出""爆发" |
| 导出图片 | 截图或另存为 `publication_trend.png` |

## 第五步：导出统一规范

- 格式必须 **PNG**（不是 JPG）
- 分辨率不能太低，至少保证放大到全屏不糊
- 所有图放入同一个文件夹 `outputs`
- 文件名不能改错，必须和上面写的完全一样（英文文件名是为了网页能自动加载）

---

## 遇到问题怎么办

| 问题 | 解决方法 |
|---|---|
| GO! 按钮灰色点不了 | 检查 Project Home 路径是否正确，关掉重开项目 |
| 跑完后一片空白 | 可能是数据没导入成功，回到第一步重新导数据 |
| 聚类效果不好（Silhouette < 0.5） | 把 Selection Criteria 里的 k 值调大（比如 30 或 50）再跑一次 |
| 标签太多重叠看不清 | 可视化窗口左边 Labels 面板调小字体，或只显示前 30 个标签 |
| 配色不满意 | 左边 Colors 面板可以改，不用管，网页端会重新排版 |
