2. **主要应用场景**
   - 研究热点识别

3. **研究范式**
   - 数据驱动 vs 问题驱动

| 要求项 | 具体内容 |

| 理解 | 文献计量学与其他计量学科的区别 |

### Lesson 3-4：数据获取

#### 核心知识点

1. **数据源选择**

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| Web of Science | 质量高、字段全、引文完整 | 学术研究首选 |
| Scopus | 覆盖广、欧洲文献多 | 补充检索 |
| CNKI | 中文文献 | 国内研究 |
| Google Scholar | 覆盖最广、但质量参差 | 初步探索 |
| OpenAlex | 免费、开放、API 友好 | 大规模分析 |

2. **检索式设计原则**

```
三层结构：
├── 对象词（研究主题）
│   └── 核心概念 + 变体 + 同义词
├── 场景词（应用领域）
│   └── 限定范围的关键词
└── 排除词（噪声过滤）
    └── 排除无关领域的术语
```

3. **检索式迭代优化**
   - v0：初步检索，评估查全率
   - v1：调整关键词，平衡查全与查准
   - v2-final：最终版本，记录变更理由

4. **数据导出规范**
   - 格式：Plain Text（全记录 + 参考文献）
   - 字段：至少包含 PT, AU, TI, SO, DE, ID, AB, C1, CR, PY, TC, DI, UT
   - 编码：UTF-8

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 掌握 | WoS 高级检索语法 |
| 应用 | 能设计三层结构检索式 |
| 产出 | 检索式设计文档（含迭代记录） |
| 文档 | `config/query.yaml` 或类似配置文件 |

### Lesson 5-6：分析方法

#### 核心知识点

1. **关键词共现分析**
   - 原理：两关键词同时出现在同一文献中，则存在共现关系
   - 矩阵：W = K.T @ K（K 为文献-关键词矩阵）
   - 用途：识别研究热点、主题聚类

2. **共被引分析**
   - 原理：两篇文献同时被第三篇文献引用，则存在共被引关系
   - 矩阵：C = A.T @ A（A 为文献-参考文献矩阵）
   - 用途：揭示知识基础、学科结构

3. **文献耦合分析**
   - 原理：两篇文献引用了相同的参考文献，则存在耦合关系
   - 矩阵：B = A @ A.T
   - 用途：发现研究前沿、相似文献

4. **合作网络分析**
   - 原理：两作者/机构共同发表文献，则存在合作关系
   - 用途：识别核心团队、合作格局

5. **聚类方法**
   - LLR 聚类：基于对数似然比，适合关键词聚类
   - 谱聚类：基于特征值分解
   - 模块度优化：Newman-Girvan 贪婪算法

6. **突现检测（Burst Detection）**
   - 算法：Kleinberg 突现检测
   - 参数：s（状态数）、γ（控制参数）
   - 用途：识别新兴趋势、热点变迁

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 理解 | 四种网络分析的原理与适用场景 |
| 掌握 | CiteSpace 参数设置 |
| 应用 | 能解读聚类结果与突现图谱 |
| 产出 | 网络图谱 + 解读文档 |

### Lesson 7-8：网络指标

#### 核心知识点

1. **节点级指标**

| 指标 | 含义 | 计算公式 |
|------|------|----------|
| Degree（度） | 节点的连接数 | d(v) = |N(v)| |
| Weighted Degree（加权度） | 考虑边权的度数 | d_w(v) = Σw(e) |
| Betweenness（中介中心性） | 经过该节点的最短路径比例 | CB(v) = Σσst(v)/σst |
| PageRank | 网页排名思想，考虑邻居重要性 | PR(v) = (1-d) + d×Σ(PR(u)/L(u)) |
| Closeness（接近中心性） | 到其他节点的平均距离倒数 | CC(v) = (n-1)/Σd(v,u) |
| Eigenvector（特征向量中心性） | 邻居重要性的加权和 | Ce(v) = 1/λ × ΣA(v,u)Ce(u) |

2. **图级指标**

| 指标 | 含义 | 解读 |
|------|------|------|
| Nodes/Edges | 节点数/边数 | 网络规模 |
| Density（密度） | 实际边数/可能边数 | 网络紧密程度 |
| Components | 连通分量数 | 网络连通性 |
| Largest Component Ratio | 最大连通分量占比 | 主流群体覆盖 |
| Communities | 社团数 | 主题/团队分化程度 |
| Modularity（模块度） | 社团结构强度 | 0.3-0.7 为合理范围 |
| Avg Clustering Coefficient | 平均聚类系数 | 局部聚集程度 |
| Diameter | 直径（最大连通分量） | 网络最远距离 |
| Avg Path Length | 平均路径长度 | 信息传播效率 |

3. **聚类评价指标**
   - Q 值（Modularity）：> 0.3 表示聚类结构显著
   - S 值（Silhouette）：> 0.5 表示聚类合理，> 0.7 为高可信

#### 课程要求

| 要求项 | 具体内容 |
|--------|----------|
| 理解 | 各指标的数学含义与适用场景 |
| 掌握 | 能计算并解读网络指标 |
| 应用 | 识别核心节点、关键文献 |
| 产出 | `network_metrics_*.csv` + `network_qc_summary.csv` |

### Lesson 17-18：案例讲解（五步法框架）

#### 五步法框架

```
Step 1: 问题定义
    ├── 明确研究问题
    ├── 确定分析目标
    └── 选择分析方法

Step 2: 数据获取
    ├── 选择数据源
    ├── 设计检索式
    ├── 导出数据
    └── 数据清洗

Step 3: 网络构建
    ├── 构建共现/共被引/耦合矩阵
    ├── 网络剪枝（可选）
    └── 参数设置

Step 4: 指标计算
    ├── 节点级指标
    ├── 图级指标
    └── 聚类评价

Step 5: 结果解读
    ├── 图谱解读
    ├── 指标分析
    ├── 结论提炼
    └── 报告撰写
```

#### 案例分析要点

1. **检索式设计**
   - 记录迭代过程
   - 说明变更理由
   - 最终检索式可复现

2. **数据清洗**
   - 去重方法（UT/DOI）
   - 筛选标准（文献类型、语言、相关性）
   - PRISMA 流程图

3. **参数固化**
   - 时间切片设置
   - 节点选择标准（g-index, Top N）
   - 剪枝方法（Pathfinder, MST）
   - 聚类算法（LLR, MI）

4. **结果呈现**
   - 图谱清晰、有图注
   - 指标表格规范
   - 结论有数据支撑

### 3.3 指标计算规范

#### 节点级指标（必须包含）

```python
# 来自 metrics.py
{
    'node': str,              # 节点名称
    'degree': int,            # 度

    'betweenness': float,     # 中介中心性
```

#### 图级指标（必须包含）

```python
# 来自 metrics.py
{
    'n_nodes': int,                      # 节点数
    'n_edges': int,                      # 边数
    'density': float,                    # 密度
    'n_components': int,                 # 连通分量数
    'largest_component_ratio': float,    # 最大连通分量占比
    'n_communities': int,                # 社团数
    'modularity': float,                 # 模块度
```

## 六、常见问题与注意事项

### 6.1 检索式设计

- ❌ 避免：过度依赖具体应用场景词，导致漏检
- ✅ 建议：使用三层结构，平衡查全与查准
- ✅ 建议：记录每次迭代的变更理由

### 6.2 数据清洗

- ❌ 避免：直接使用检索结果，未做去重
- ✅ 建议：UT + DOI 双重校验
- ✅ 建议：明确筛选标准，记录排除原因

### 6.3 参数设置

- ❌ 避免：使用默认参数，未说明选择理由
- ✅ 建议：参数固化文档，可追溯
- ✅ 建议：对比不同参数的结果稳定性

### 6.4 结果解读

- ❌ 避免：只描述现象，不解释原因
- ❌ 避免：结论超出数据支撑范围
- ✅ 建议：结合领域知识解读
- ✅ 建议：指出研究的局限性

### 6.5 可复现性

- ❌ 避免：代码无注释、依赖无版本
- ❌ 避免：操作步骤缺失
- ✅ 建议：使用模板提供的标准结构
- ✅ 建议：README 包含完整运行命令
