# 经验

## 项目结构

- 根目录和 `Transformer-Semiconductor-Bibliometrics/` 各有一个 Git 仓库，操作前确认在哪个仓库
- 子仓库分支是 `main`（非 master）

## 环境

- npm 网络不稳时加 `--registry=https://registry.npmmirror.com`
- Tailwind CSS v4 + Vite 必须装 `@tailwindcss/vite`，删掉 `postcss.config.js`
- lucide-react 没有 `Github` 图标，用 `ExternalLink` 替
- 分支切换后 node_modules 会坏，每次切回 main 要重跑 `npm install` 再 `npm run build`

## rm 红线

- **`rm -rf` 前必须先 `ls -la` 确认目标** — 尤其 `rm -rf * .*` 会删掉 `.git`，仓库就没了
- 用精确路径，别用通配符批量删

## GitHub Pages 部署

- 不要在 gh-pages 分支用 `git add -A`，会混入 node_modules
- 网页是独立 HTML（CDN Tailwind + Lucide），无需构建：
  1. `git checkout gh-pages`
  2. `cp 桌面/网页.html index.html`
  3. `rm -rf assets`（如果旧版有构建产物）
  4. `git add index.html favicon.svg icons.svg`
  5. `git commit` → `git push origin gh-pages`
- 网页 URL：`https://jyz2744153437-max.github.io/Transformer-Semiconductor-Bibliometrics/`

## 网站

- 源代码即桌面的 `网页.html`，独立 HTML（非 React），直接用浏览器打开
- old site/ 目录是 React/Vite 废案，保留供参考但不再维护
- 7 张 CiteSpace 图的 PNG 文件名：`publication_trend.png`、`keyword_cooccurrence_network.png`、`keyword_cluster.png`、`co_citation_network.png`、`co_citation_cluster.png`、`burst_timeline.png`、`timeline_view.png`
